# Playtest 001: Small Utility vs Script Kiddie

## Purpose

This first playtest validates whether the ICS-WATER edition is fun, teachable, and easy to run at a tabletop event.

## Matchup

- **Blue Team:** Small Utility
- **Red Team:** Script Kiddie

## Training goals

Players should learn:

- Why unsupported HMI systems are risky.
- Why default credentials and exposed remote access matter.
- How network segmentation reduces spread.
- Why backups, manual override, and local operators are important.
- How simple attacks can still create operational impact.

## Setup

- Each side uses the starter deck list in `data/editions/ICS-WATER/decks/small-utility-vs-script-kiddie.deck.csv`.
- Each player starts with 20 SCP.
- Each player draws 7 cards.
- Use the standard Cyber Realm turn structure.

## Scenario option

Add **Water Tower Overflow** as the scenario objective.

Red Team wins the scenario if Water Tower and Remote Access Pathway are both damaged before turn 6.

Blue Team wins the scenario if Water Tower survives through turn 6.

## What to observe

During playtesting, record:

- Did players understand card effects without explanation?
- Did the game feel too slow or too fast?
- Did Blue Team have enough ways to recover?
- Did Red Team have enough ways to chain attacks?
- Were any cards too strong, too weak, or confusing?
- Did players learn something useful about water-sector cybersecurity?

## Balance notes

Expected beginner match length: 15-25 minutes.

Red Team should feel threatening but noisy.

Blue Team should feel under-resourced but capable of clutch recovery.
