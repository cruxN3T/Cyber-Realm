# HTTP vs Plain Web Protocol Naming

## Decision

HTTP is acceptable to use as a card name because it is a standard protocol term, not a vendor/tool brand like a specific scanner or commercial product.

## Recommendation

For clarity, use:

> HTTP

instead of:

> Plain Web Protocol

## Why

AWWA and security players are more likely to recognize HTTP. The card can still teach the same lesson: exposed or plaintext web services can increase visibility and risk.

## Suggested card treatment

- Card name: HTTP
- Category: Protocol
- Realm: Internet or Application depending on the card role
- Phase: Reinforce or Exposure/Risk tag
- Cost: 0 if it represents inherited exposure/risk

## Suggested card text

Attach to an Application Asset. That Asset costs 1 less to play but takes +1 damage from Internet Attacks.

## Flavor text

Easy access has a cost.

## Note

If the card represents the port rather than the protocol, use Port 80. If it represents the protocol behavior, use HTTP.
